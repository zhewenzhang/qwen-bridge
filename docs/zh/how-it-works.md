# 工作原理

## 架构设计

```
┌─────────────────┐     MCP 协议        ┌─────────────────┐     后台启动进程      ┌─────────────────┐
│   Claude Code   │ ──────────────────→  │   AutoClaude    │ ────────────────────→  │   AI 编程 CLI    │
│   （规划者）     │                      │   （派发器）     │                        │   （执行者）     │
│                 │ ←──────────────────  │                 │ ←────────────────────  │                 │
│  策略           │    流程报告          │  验证           │    结果日志            │  文件操作       │
│  架构设计       │    Token 节省        │  通知           │    摘要报告            │  Git 提交       │
│  验证结果       │                      │  成本追踪       │                        │  构建部署       │
└─────────────────┘                      └─────────────────┘                        └─────────────────┘
```

## 角色分工

| 角色 | 系统 | 职责 | Token 来源 |
|------|------|------|-----------|
| **规划者** | Claude Code | 读取代码库、设计架构、编写任务文件、验证结果 | Claude API |
| **派发器** | AutoClaude | 验证任务、启动 Agent、捕获输出、生成报告 | 无（本地运行） |
| **执行者** | AI CLI Agent | 编辑文件、执行 Git 操作、构建、测试 — 所有实现工作 | Agent 自身的 Token 池 |

## 派发流程（CLI Agent）

```
1. Claude 调用 dispatch_task("MY_TASK.md")
        │
2. AutoClaude 读取任务文件
        │
3. 生成 _summary.md 头部（元数据 + 角色表）
        │
4. 用格式化指令包装任务
        │
5. 写入 .bat 文件：type task.txt | qwen -y > result.log 2>&1
        │
6. 通过 cmd.exe start /min 启动（后台静默，MCP 重启不影响）
        │
7. 立即返回 "✅ 已派发" 给 Claude
        │
8. Agent 在后台以 YOLO（自动批准）模式执行
        │
9. 输出捕获到 _result.log
        │
10. Claude 调用 get_task_report() → 自动完成 _summary.md
```

## Token 经济学

Claude 只在规划阶段消耗 Token（每次任务约 5K-8K）。繁重的执行工作（代码编辑、Git 操作、构建）由 Agent 使用自己的 Token 池完成。每次任务可节省 **60-80% 的 Claude Token**。

使用 `get_savings_report` 查看你的累计节省。
